## Summary by Category

**Abstraction isolation (2):** The HID protocol layer and CBOR value handling are both mixed into the files that use them rather than living in focused modules.

**Duplicate logic (3):** PIN token acquisition, confirmation prompts, and bootloader signature selection all appear verbatim in 2–3 places each.

**Idiomatic Rust (2):** Several functions are 150–280 lines and mix abstraction levels; two `unsafe` blocks exist purely because a safe API was not used.

**Security (3):** One `unwrap()` in HMAC creation can panic, the all-zero AES IV is spec-mandated but undocumented (and looks like a bug), and attestation verification only checks certificate fingerprints with no chain/date/revocation validation.

**Tests (2):** `program.rs` and `aux.rs` have no tests; all HID-using commands are untestable without a physical device due to the lack of a mock/trait abstraction.
